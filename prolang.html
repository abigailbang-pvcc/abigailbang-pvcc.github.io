      Chapter 6: Programming Languages

      • Machine language - Machine language is the lowest-level programming language consisting of binary code directly executed by a computer's central processing unit (CPU).

      • Debugging - Identifying and altering errors within the program

      • Assembler - turns mnemonic expressions into machine language, allowing humans to use mneumonic form

      • Assembly language - collective tern which refers to mneumonic systems representing programs

      • Translator - A program capable of translating high-level primitives into machine-language programs after they are identified. They had similarites to second-generation assemblers.

      • Compiler - A translation program which compiled multiple machine instructions to create short sequences. They simulated activity that a high-level primitive would request.

      • Interpreter - Capable of setting up third-generation languages, it was an alternite option for translators.

      • Cross-platform software - Separate from the operating system, it is software which is executable trhoughout the network. It is also separate from the hardware of the machine.

      • Natural language - Languages which form gradually over time, without formal grammatical analysis.

      • Formal language - Defined by grammars.

      • Machine independence - It was largly achieved due to third-generation languages. High-level programming languages offer the programmer more abstractions

      • High-level programming language - Offering a greater number of abstractions to the programmer, they are easier to read and write.

      • Object-oriented programming (OOP) - A programmng paradigm which is the object-oriented paradigm. Objects are collected by the software system, and they are able to perform actions relating to themselves. They request actions from other objects, and interact to complete problems.

      • Scripting language - Scripting languages constitue a subset of imperative programming languages. They are commonly utilized for administrative tasks rather than the development of complex programs.

      • Data types:
        • Integer - Numeric data containing whole numbers
        • Float - Additionally called real, it is numeric data which can contain non whole number values.
        • String - A string is a sequence of characters, treated as a single data type, commonly used to represent and manipulate textual information.

      • Primitive data type - A primitive data type in programming is a fundamental, basic value type like integers, floats, or characters.

      • Data structure - The way data is organized, and it is frequently linked with program variables.

      • Python collection data structures:
        • List - A list is a flexible and ordered container for storing and manipulating data.
        • Tuple - A tuple is an immutable and ordered collection of elements, often used to group related data. It cannot be altered by the users of the program.
        • Set - A set is an unordered collection of modifiable elements.
        • Dictionary - A mutable and unordered data type designed to store key-value pairs, where each key must be unique.

      • Function - Instructions for executing a task, which can be used as an abstract tool.

      • Local variable - Declared withen a function, they are not able to be referenced outside on that function.

      • Global variable - Not limited to just one section of the program, and can be used outside of one function. They span the whole program.

      • Source program - The original form of a program.

      • Object program - The translated version of the source program.
