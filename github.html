      • Git/Github terms:

      • Repository
        • Repositories store your projects in Github. You can name them, and made them open source.

      • Commit
        • A commit is equvalent to saving your work, such as new changes to a project.

      • Fork
        • A fork means a repository which shares settings and code with an original repository which was created first.

      • Push
        • A push updates local changes to remote branches.

      • Pull requets
        • A pull request is how you can recieve the changes created by a push. It updates your version of the project from the remote branch.

      • Workflows
        • Workflows are automated processes which can run jobs. They can be triggered manually or be on a schedule.

      • Issues
        • Issues are a way of tracking work. They can help you track your work using cross-referencing of time worked.

      • Raw button
        • The raw button lets you open the file in its raw form. Formatting disappears.

      • Blame button
        • It shows the time when a specific section of code was altered.
